# GeneVault API

**GeneVault API** — это бэкенд-сервис для хранения и автоматического биоинформатического анализа генетических последовательностей. Проект предоставляет инструменты для безопасного хранения ДНК-данных и автоматического расчета их ключевых физико-химических свойств.

## Решение бизнес-задачи
Проект не просто сохраняет строки (CRUD), а решает реальную задачу предварительного анализа данных:
* **Автоматические вычисления**: При добавлении новой ДНК-последовательности система автоматически рассчитывает GC-состав (GC content), температуру плавления (Melting Temperature, Tm) и молекулярную массу.
* Данные вычисления инкапсулированы в модуле `bio_calc.py` и доступны сразу после загрузки сиквенса.

## Технологический стек
* **Фреймворк**: FastAPI
* **База данных**: SQLite + SQLAlchemy (ORM)
* **Валидация**: Pydantic (включая RegEx-валидацию нуклеотидных последовательностей)
* **Аутентификация**: OAuth2 с JWT-токенами, хеширование паролей (Passlib/Bcrypt)
* **Контейнеризация**: Docker & Docker Compose
* **Тестирование**: Pytest + TestClient
* **Линтинг**: Pylint (Оценка: 8.54/10)

## Архитектура БД и CRUD
Реализована полноценная реляционная база данных (3 связанные таблицы):
1. **Users** (Пользователи)
2. **Sequences** (Генетические последовательности, привязаны к пользователю `owner_id`)
3. **AnalysisResults** (Результаты анализа, связь 1-к-1 с `Sequences`)

Для сущностей реализован полный набор CRUD-операций (всего **12 эндпоинтов**), включая создание, чтение (списком и детально), обновление (переименование) и каскадное удаление.

## Авторизация и валидация
* **JWT / OAuth2**: Защита эндпоинтов реализуется через механизм `OAuth2PasswordBearer`. Большинство маршрутов требуют передачи Bearer-токена.
* **Pydantic**: Жесткая валидация входящих данных. Например, `raw_sequence` принимает строго символы нуклеотидов (A, T, G, C) благодаря регулярному выражению `^[ATGCatgc\s]+$`.

## Установка и запуск

### Вариант 1: Запуск через Docker (Рекомендуемый)
Убедитесь, что у вас установлены Docker и Docker Compose.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Dudkanist/FastAPI-project.git
   cd FastAPI-project
   ```
2. Переименуйте `.env.example` в `.env` и укажите ваши настройки:
   ```bash
   cp .env.example .env
   ```
3. Запустите контейнеры:
   ```bash
   docker-compose up --build
   ```
Приложение будет доступно по адресу: `http://localhost:8000`

### Вариант 2: Локальный запуск
1. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Linux/Mac
   venv\Scripts\activate     # Для Windows
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Выполните запуск сервера:
   ```bash
   uvicorn app.main:app --reload
   ```

Проект покрыт автоматическими тестами с использованием `pytest` и `TestClient` от FastAPI, тестирующими ключевую логику и эндпоинты.
